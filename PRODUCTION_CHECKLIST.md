## 결제 시스템

### PG사(Payment Gateway) 선택
- [ ] **주요 PG사 검토**
  - 국내: 이니시스, KG이니시스, 나이스페이먼츠, KCP, 토스페이먼츠, 페이플
  - 해외: 스트라이프(Stripe), 페이팔(PayPal)
  - 수수료율, 정산 주기, 지원 결제수단 비교

- [ ] **결제 수단 지원**
  - 신용카드/체크카드
  - 계좌이체
  - 가상계좌
  - 간편결제 (네이버페이, 카카오페이, 토스, 페이코 등)
  - 해외카드 (다이빙 고객 중 외국인 많음)

### 결제 보안 (PCI DSS 준수)
- [ ] **카드 정보 처리**
  - ⚠️ **절대 카드번호를 서버에 저장하지 말 것**
  - PG사 제공 토큰화(Tokenization) 사용
  - PCI DSS Level 준수 (PG사가 대부분 처리)

- [ ] **통신 보안**
  - SSL/TLS 인증서 필수 (HTTPS)
  - TLS 1.2 이상 사용
  - 결제 페이지는 별도 도메인 또는 서브도메인 검토

- [ ] **결제 프로세스 보안**
  - 결제 요청 시 서버 검증 필수
  - 결제 금액 조작 방지 (클라이언트에서 금액 전송 금지)
  - CSRF 토큰 사용
  - 중복 결제 방지 (idempotency key)

### 결제 플로우 설계
```
사용자 주문 생성
  ↓
서버에서 주문 정보 저장 (PENDING 상태)
  ↓
PG사 결제 페이지 호출 (금액 서버에서 검증)
  ↓
결제 완료 후 Callback/Webhook
  ↓
서버에서 결제 결과 검증 (금액, 주문번호 확인)
  ↓
주문 상태 업데이트 (PAID)
  ↓
사용자 확인 페이지 + 이메일 발송
```

- [ ] **결제 후 처리**
  - 결제 결과 웹훅(Webhook) 처리
  - 결제 금액 및 주문 정보 재검증
  - 주문 상태 관리 (PENDING → PAID → CONFIRMED → COMPLETED)
  - 영수증 발급
  - 확인 이메일 자동 발송

- [ ] **환불 및 취소**
  - 부분 환불 지원
  - 환불 요청 시 PG사 API 호출
  - 환불 내역 로그 저장
  - 전자상거래법 상 청약철회권 처리 (7일 이내)

## 보안 요구사항

### 필수 보안 조치
- [ ] **HTTPS 적용**
  - Let's Encrypt 또는 유료 SSL 인증서
  - HTTP → HTTPS 자동 리다이렉트
  - HSTS 헤더 설정

- [ ] **XSS (Cross-Site Scripting) 방어**
  - 사용자 입력 sanitization
  - React는 기본적으로 XSS 방어하지만, dangerouslySetInnerHTML 사용 금지
  - Content Security Policy (CSP) 헤더 설정

- [ ] **CSRF (Cross-Site Request Forgery) 방어**
  - CSRF 토큰 사용
  - SameSite 쿠키 속성 설정

- [ ] **SQL Injection 방어**
  - ORM 사용 (Sequelize, TypeORM, Prisma 등)
  - Prepared Statement 사용
  - 사용자 입력 직접 쿼리에 삽입 금지

- [ ] **인증 및 인가**
  - JWT 또는 세션 기반 인증
  - 비밀번호 해싱 (bcrypt, argon2)
  - 비밀번호 복잡도 정책
  - 다중 인증(2FA) 검토 (관리자 필수)
  - Rate Limiting (무차별 대입 공격 방지)

- [ ] **API 보안**
  - API 키 관리 (환경 변수, Secrets Manager)
  - API Rate Limiting
  - API 인증 (OAuth 2.0, API Key)
  - CORS 정책 설정

### 개인정보 보호
- [ ] **데이터 암호화**
  - 개인정보 DB 암호화 (이름, 전화번호, 이메일)
  - 비밀번호 단방향 해싱
  - 전송 중 암호화 (TLS)

- [ ] **접근 제어**
  - 관리자 페이지 IP 화이트리스트
  - 역할 기반 접근 제어 (RBAC)
  - 최소 권한 원칙

- [ ] **개인정보 보관 기간**
  - 회원 탈퇴 시 개인정보 즉시 삭제
  - 법적 보관 의무 데이터 별도 관리
  - 장기 미이용자 개인정보 파기 (1년)

## 법령 및 규제 준수

### 필수 법률 준수
⚠️ **전문 변호사/법무법인 검토 필수**

#### 1. 개인정보보호법
- [ ] **개인정보 수집 및 이용 동의**
  - 수집 목적, 항목, 보유 기간 명시
  - 필수/선택 항목 구분
  - 동의 체크박스 (전체 동의 금지, 개별 동의)

- [ ] **개인정보 처리방침 페이지**
  - 수집하는 개인정보 항목
  - 개인정보의 보유 및 이용 기간
  - 개인정보 제3자 제공 (있는 경우)
  - 개인정보 처리 위탁 (있는 경우)
  - 정보주체의 권리와 행사 방법
  - 개인정보 보호책임자 연락처

- [ ] **개인정보 보호책임자 지정**
  - 이름, 연락처 공개

#### 2. 전자상거래 등에서의 소비자보호에 관한 법률
- [ ] **통신판매업 신고**
  - 관할 구청/시청에 통신판매업 신고 필수
  - 신고번호 웹사이트에 표기

- [ ] **사업자 정보 공개**
  - 상호, 대표자 이름
  - 사업자등록번호
  - 통신판매업 신고번호
  - 주소, 연락처
  - 호스팅 서비스 제공자

- [ ] **이용약관**
  - 서비스 이용 약관
  - 청약철회 및 환불 정책
  - 배송 정책 (해당 시)

- [ ] **청약철회권 (7일 이내)**
  - 구매일로부터 7일 이내 환불 가능
  - 예외 사항 명시 (디지털 콘텐츠, 서비스 이미 제공 등)
  - ⚠️ 다이빙 투어는 "용역" 또는 "서비스"에 해당
    - 예약 시점과 서비스 제공 시점 구분
    - 취소 정책 명확히 명시 (예: 7일 전 무료, 3일 전 50%, 당일 불가 등)

#### 3. 전자금융거래법
- [ ] **전자금융거래 이용약관**
  - PG사 제공 표준 약관 사용 가능

- [ ] **거래 내역 제공**
  - 사용자가 결제 내역 조회 가능
  - 최소 5년간 보관

- [ ] **보안성 확보 조치**
  - 거래 내용 암호화
  - 사용자 인증

#### 4. 표시·광고의 공정화에 관한 법률
- [ ] **과장/허위 광고 금지**
  - PADI 인증, 시설 정보 등 정확히 표기
  - 리뷰 조작 금지

#### 5. 소비자기본법
- [ ] **부당한 조항 금지**
  - 일방적으로 불리한 약관 금지

## 개발 우선순위

### Phase 1: MVP (필수)
